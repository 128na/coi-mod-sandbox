### 3. FastFlatConveyor のグラフィック変更

#### カスタムマテリアルの試み（失敗）

`TransportProto.Gfx` の `MaterialPath` フィールドは **readonly**。
コンストラクタで新規 `Gfx` を作成し `materialPath` を指定したが、ゲーム内で見た目は変わらなかった。

```csharp
graphics: new TransportProto.Gfx(
    ...
    materialPath: "Assets/ExampleMod2/Transports/FastFlatConveyor_mat.mat",
    ...
)
```

**推定原因**: ゲームが mod の AssetBundle からマテリアルをロードする仕組みに `IAlternativeAssetLoader` が関与している可能性。
削除した `EditorAssetsLoaderModMb.cs` がそのブリッジだったかもしれない。

#### T3 グラフィック流用（成功）

```csharp
graphics: t3.Graphics
```

T3（赤）の外観で FastFlatConveyor が表示されることを確認。✅

## 次のステップ

- カスタムカラーのマテリアルを適用できるようにする

## 得られた知見

- `TransportProto.Gfx` のフィールドはすべて **readonly** → コンストラクタで新規作成が必要
- `graphics: t3.Graphics` のように既存の Gfx をそのまま渡すことは可能
- `[MaFi]` メニューは Mafi DLL がエディタに読み込まれることで有効になる
- Unity 6000.0.66f1 以外（6000.4.x 等）では AssetBundle の互換性問題が起きる可能性がある
