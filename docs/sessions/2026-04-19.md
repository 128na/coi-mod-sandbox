## ハマったポイント

### IoPortShapeProto 1対1 制約によるクラッシュ

**エラー**: `ArgumentException: Adding already existing element key=IoPortShape_FlatConveyor`

`LayoutEntityBlueprintGenerator` が `IoPortShape → TransportProto` 辞書を構築するため、
同じ `IoPortShapeProto` を複数 Transport で共有するとクラッシュする。

**試行1**: 新しい `IoPortShapeProto` を作成して登録 → クラッシュは回避できたが倉庫に接続不可  
**試行2**: `t3.TierData.SetNextTierIndirect(proto, false)` → `PreviousTier` が設定されず効果なし  
**試行3**: 新しい `IoPortShapeProto` を ProtosDb 未登録で使用 → 倉庫接続不可（インスタンス同一性確認）  
**試行4（成功）**: 元の portShape インスタンス + `t3.Upgrade.SetNextTier(proto, overwrite: false)`

### ポート接続はインスタンス同一性

建物は特定の `IoPortShapeProto` インスタンスを保持している。
`LayoutChar` が同じでも別インスタンスでは接続できない。

### SetNextTierIndirect vs SetNextTier

- `TierData.SetNextTierIndirect` → indirect tier のみ設定、`Upgrade.PreviousTier` には影響しない
- `Upgrade.SetNextTier` → アップグレードチェーンに登録、`PreviousTier.IsSome` が true になる
- `LayoutEntityBlueprintGenerator` がフィルタするのは `Upgrade.PreviousTier.IsSome` のもの

## 3. ProtoDumpMod（独立 mod）

ゲーム内プロトタイプデータを JSON ファイルに出力する独立 mod を作成。

- `ProtoDumpMod/` を新規プロジェクトとして作成
- `PrototypesDb.Filter<T>()` で全プロト列挙（`All<T>()` は空を返したため Filter を使用）
- 起動時に `%APPDATA%\Captain of Industry\Mods\ProtoDumpMod\` へ JSON ファイルを出力
- `transports.json`, `products.json`, `machines.json`, `recipes.json`, `io_port_shapes.json`, `research.json` を取得
- 取得データを `docs/game-data/*.json` にコピーして保存

### `All<T>()` が空を返す問題

`PrototypesDb.All<T>()` が常に空を返した。`Filter<T>(_ => true)` では正常に取得できた。
原因不明（型の完全一致 vs 継承の扱いの違いの可能性）。

4. Unity が DLL を認識しないため `.meta` ファイルを手動生成
5. `EditorAssetsLoaderModMb.cs` がコンパイルエラーを引き起こすため削除
